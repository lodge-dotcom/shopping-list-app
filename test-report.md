## 상세 테스트 결과

### TC-01 · 아이템 추가 (Enter 키)
- **동작:** 입력창에 "사과", "우유" 입력 후 Enter 키 제출
- **기댓값:** 각 아이템이 목록에 추가되고 카운트 반영
- **결과:** ✅ 통과
- **확인값:** 헤더 "0 / 2 완료", 목록에 사과·우유 표시

### TC-02 · 아이템 추가 (추가 버튼 클릭)
- **동작:** 입력창에 "빵" 입력 후 추가 버튼 클릭
- **기댓값:** 빵이 목록 최상단에 추가됨
- **결과:** ✅ 통과
- **확인값:** 헤더 "0 / 3 완료", 푸터 "총 3개 · 완료 0개"

### TC-03 · 빈 입력값 방어
- **동작:** 입력창이 빈 상태에서 추가 버튼 클릭
- **기댓값:** 아이템이 추가되지 않음
- **결과:** ✅ 통과
- **확인값:** 목록 아이템 수 3개 유지 (변화 없음)

### TC-04 · 체크박스로 완료 처리
- **동작:** 빵, 우유 체크박스 클릭
- **기댓값:** 체크박스 파란색으로 변경, 텍스트에 취소선 표시
- **결과:** ✅ 통과
- **확인값:** `done` 아이템 2개, 헤더 "2 / 3 완료", 푸터 "총 3개 · 완료 2개"

### TC-05 · 체크 토글 (완료 → 미완료)
- **동작:** 완료 상태의 빵 체크박스 재클릭
- **기댓값:** 미완료 상태로 복원
- **결과:** ✅ 통과
- **확인값:** `done` 아이템 1개, 헤더 "1 / 3 완료"

### TC-06 · 미완료 필터
- **동작:** "미완료" 탭 클릭
- **기댓값:** done=false 아이템만 표시
- **결과:** ✅ 통과
- **확인값:** 표시 목록 `["빵", "사과"]` (우유 숨김), 활성 필터 "미완료"

### TC-07 · 완료 필터
- **동작:** "완료" 탭 클릭
- **기댓값:** done=true 아이템만 표시
- **결과:** ✅ 통과
- **확인값:** 표시 목록 `["우유"]`, 활성 필터 "완료"

### TC-08 · 전체 필터 복귀
- **동작:** "전체" 탭 클릭
- **기댓값:** 모든 아이템 다시 표시
- **결과:** ✅ 통과
- **확인값:** 표시 목록 `["빵", "우유", "사과"]`, 활성 필터 "전체"

### TC-09 · 개별 아이템 삭제
- **동작:** 사과 행의 ✕ 버튼 클릭
- **기댓값:** 사과만 목록에서 제거, 나머지 유지
- **결과:** ✅ 통과
- **확인값:** 표시 목록 `["빵", "우유"]`, 푸터 "총 2개 · 완료 1개"

### TC-10 · 완료 항목 일괄 삭제
- **동작:** "완료 항목 삭제" 버튼 클릭
- **기댓값:** done=true 아이템(우유)만 삭제, 미완료 아이템(빵) 유지
- **결과:** ✅ 통과
- **확인값:** 표시 목록 `["빵"]`, 푸터 "총 1개 · 완료 0개"

### TC-11 · localStorage 영속성 (페이지 새로고침)
- **동작:** 페이지 새로고침 (`http://localhost:8765/shopping-list.html` 재접속)
- **기댓값:** 이전 상태가 복원됨
- **결과:** ✅ 통과
- **확인값:** localStorage에 `[{"id":...,"text":"빵","done":false}]` 존재, 빵 아이템 정상 복원
